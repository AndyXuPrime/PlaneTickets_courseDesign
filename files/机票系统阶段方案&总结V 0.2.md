## 第三阶段：Admin 业务逻辑实现思路

在微服务中，Admin 服务想要统计“热门航线”或“销售总额”，不能直接写 SQL 查 `tickets` 表（跨库了）。

**你有两个选择：**

1.  **方案 A（标准微服务）**：
    *   在 `order-service` 的 Controller 里写好统计接口（如 `/api/bookings/stats/revenue`）。
    *   `admin-service` 通过 `OrderFeignClient` 调用这个接口，拿到数据直接返回给前端。
    *   **优点**：架构清晰。
    *   **缺点**：`order-service` 负担重。

2.  **方案 B（课程设计捷径 - 读写分离）**：
    *   让 `admin-service` 也连接 `planetickets` 数据库。
    *   但是只写 **只读** 的 Repository（只用 `@Query` 查报表）。
    *   **优点**：写报表 SQL 非常方便，不用跨服务调来调去。
    *   **缺点**：理论上微服务不共享数据库，但在做报表/BI分析场景下，这种“旁路读取”是可以接受的。

**推荐**：
为了省事且代码好写，你可以采用 **方案 B**。
即：在 `admin-service` 里引入 JPA，配置数据库，把原来的 `TicketRepository` 里的统计 SQL 搬过来。

### 4. 🛠️ 踩坑记录与解决方案 (Troubleshooting Knowledge Base)

这些是你解决过的技术难题，也是答辩时的亮点。

#### A. Maven 与 依赖管理
*   **问题**：Gateway 启动报错 `No plugin found for prefix 'spring-boot'`。
    *   **解决**：在 Gateway 的 `pom.xml` 中补全 `<build><plugins>...</plugins>`。
*   **问题**：Gateway 报错 `package com.bighomework.common... does not exist`。
    *   **原因**：Common 模块修改后未安装到本地仓库。
    *   **解决**：在根目录执行 **`mvn clean install -DskipTests`**。

#### B. Nacos 连接与配置
*   **问题**：`Connection refused: ...:9848`。
    *   **原因**：Nacos 2.x 客户端需要连接 gRPC 端口 (9848)。
    *   **解决**：确保 Nacos 正常启动且端口未被防火墙拦截。
*   **问题**：`Unknown database 'plane_ticket_db'`。
    *   **原因**：本地数据库名是 `planetickets`，配置文件写错了。
    *   **解决**：修改 Nacos 中 `flight-service.yaml` 的 JDBC URL。

#### C. 跨模块配置依赖 ("吃大户"问题)
*   **问题**：Gateway 和 Flight 启动报错 `Could not resolve placeholder 'app.jwt.secret'`。
    *   **原因**：Common 模块的 `JwtTokenProvider` 需要密钥，但引用它的服务没有在配置里提供。
    *   **解决**：在 Nacos 的 `gateway-service.yaml` 和 `flight-service.yaml` 中 **追加 JWT 配置块**。

#### D. 技术栈冲突
*   **问题**：Gateway 引入 Common 后被迫引入 JPA，导致启动检查数据库失败。
    *   **解决**：在 `GatewayApplication` 注解中添加 `exclude = {DataSourceAutoConfiguration.class}`。
